# Memory Bank — guía rápida

Propósito:
- Almacenar el contexto del proyecto entre sesiones de trabajo. Debe leerse al inicio de cada tarea.

Estructura requerida (core files):
- `projectbrief.md` — visión resumida del proyecto.
- `productContext.md` — por qué existe el producto y prioridades.
- `activeContext.md` — en qué estamos trabajando ahora.
- `systemPatterns.md` — arquitectura y patrones del sistema.
- `techContext.md` — tech stack y setup.
- `progress.md` — estado y bloqueos.

Uso:
1. Leer todos los archivos antes de comenzar a trabajar.
2. Actualizar `activeContext.md` y `progress.md` al finalizar cambios significativos.
3. El agente automatizado (Copilot) depende de estos archivos después de resets.
